Visão Geral Os Ataques Como Detecta Resultados Reais

🛡️ Lab de Detecção de Intrusão

Suricata IDS + Ataques reais — tudo na mesma máquina Linux

🛡️

Suricata — Defensor

IDS open-source que monitora o tráfego de rede em tempo real e gera alertas quando detecta comportamento suspeito.

⚔️

Nmap / Ping / Curl — Invasor

Ferramentas reais de pentest executando port scan, flood ICMP e requisições HTTP maliciosas contra a própria máquina.

Como funciona o lab

Tudo roda em uma única máquina. O atacante e o defensor são o mesmo computador.

O Suricata monitora a interface lo (loopback) em modo pcap — assim captura tráfego interno que vai de 127.0.0.1 para 127.0.0.1.

Quando um ataque gera tráfego suspeito, o Suricata compara com suas 8 regras personalizadas e gera alertas em tempo real nos logs.

Objetivo: demonstrar que um IDS detecta padrões de ataque pelo comportamento do tráfego — não pelo conteúdo dos arquivos ou pelo antivírus.

⚔️ Os 4 Ataques

Técnicas reais de reconhecimento usadas em pentests

ATAQUE 1

Port Scan SYN — Reconhecimento de portas

nmap -sS 127.0.0.1

Envia pacote SYN para centenas de portas sem completar a conexão. O objetivo é descobrir quais portas estão abertas sem deixar rastro no log do servidor alvo.

✓ Detectado — SID 9000001
ATAQUE 2

Scan Agressivo — Identificação do sistema

nmap -A 127.0.0.1

Detecta sistema operacional, versões de serviços e roda scripts automatizados. Gera muito mais tráfego — inclui scans NULL e XMAS internamente.

✓ Detectado — SID 9000002, 9000004, 9000005
ATAQUE 3

ICMP Flood — Negação de serviço

ping -f -c 100 127.0.0.1

Envia 100 pings em modo flood (velocidade máxima — 100 pacotes em ~2ms). Simula ataque de negação de serviço por volume.

✓ Detectado — SID 9000010
ATAQUE 4

HTTP Suspeito — Scanner e Path Traversal

curl -A "sqlmap/1.0" http://127.0.0.1:8080/ curl http://127.0.0.1:8080/etc/passwd

Primeira requisição imita o scanner SQLMap pelo User-Agent. Segunda tenta acessar /etc/passwd diretamente — técnica de Path Traversal / LFI.

✓ Detectado — SID 9000020, 9000022

🔍 Como o Suricata Detecta

Duas abordagens: volume de pacotes e assinatura de conteúdo

Abordagem 1 — Threshold (volume)

Para port scan e ICMP flood, a regra conta quantos pacotes chegam em quanto tempo. Um único SYN é normal. 20 SYN em 3 segundos é port scan.

# Regra de port scan:
alert tcp any any -> any any (
  msg:"LAB PORT SCAN SYN scan";
  flags:S; ← só pacotes SYN
  detection_filter: track by_src,
    count 20, seconds 3; ← 20 em 3s = alerta
  sid:9000001;
)

Abordagem 2 — Assinatura (conteúdo)

Para ataques HTTP, a regra procura strings específicas no conteúdo do pacote — User-Agent, URI, etc.

# Regra SQLMap:
alert http any any -> any any (
  msg:"LAB HTTP SQLMAP user-agent";
  http.user_agent; content:"sqlmap"; nocase;
  ← encontrou "sqlmap" no header = alerta imediato
  sid:9000020;
)
